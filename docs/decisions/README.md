# Architecture Decision Records

One file per decision: `NNNN-short-title.md` (e.g. `0001-tsvector-trigger-migration.md`).

Use this format:

```markdown
# NNNN. Title

Date: YYYY-MM-DD
Status: proposed | accepted | superseded by NNNN

## Context
What forced this decision.

## Decision
What was decided.

## Consequences
What this makes easier or harder going forward.
```

See `docs/SDS.md` Section 12 for the open decisions expected to land here as tickets implement
them (tsvector trigger migration, version retention limit, share token storage strategy).
