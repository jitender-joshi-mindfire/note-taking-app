# changes/

Active, in-progress per-ticket proposals (`changes/AB-xxxx-short-name/`). Each contains
`proposal.md`, `specs/<capability>/spec.md` (delta), `design.md`, and `tasks.md`, created by the
`/spec`, `/plan`, and `/tasks` slash commands in that order.

Empty until the first ticket runs `/spec AB-xxxx`. A change moves out of here into `archive/`
via `openspec archive <change-name>` once its ticket is complete (Rule 18 — required before
raising a PR).
